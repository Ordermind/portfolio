Statistics application

A callcenter needed to retrieve statistics of for example the number of calls, the number of “yes” responses and the number of “no” responses. They wanted to be able to view the statistics in a table and save it to an Excel file, but they also wanted the statistics to be sent automatically every day to a partner.

Since the project involves sensitive data I selected Drupal as a foundation so that we could benefit from Drupal’s high security and get a battle tested user system out of the box. With the help of Drupals’ internal api it was a trifle to pull the right information from the callcenter database and present it in the desired way. I also set up a cron job which sent the statistics by SFTP to their partner.
